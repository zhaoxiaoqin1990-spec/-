Claude Code 常用命令使用教程

从入门到熟练 —— 掌握 Claude Code 的高效工作流

1斜杠命令 Slash Commands

在对话输入框中直接输入 / 即可唤出命令菜单，常用命令如下：

命令作用
/help查看帮助文档
/clear清空当前对话历史
/compact压缩历史对话以节省上下文
/model切换模型（Opus / Sonnet / Haiku）
/config打开配置界面
/status查看当前会话状态
/cost查看本次会话的 token 消耗
/login / /logout登录 / 登出账号
/init在项目中生成 CLAUDE.md 记忆文件
/memory编辑记忆文件
/review让 Claude 审查当前代码改动
/pr-comments查看 GitHub PR 评论
/bug上报 bug
/doctor诊断 Claude Code 运行环境
/fast切换快速输出模式

2命令行启动参数

在终端使用 claude 命令启动，支持多种参数：

claude                     # 交互模式
claude "帮我写单元测试"      # 直接发送一个提示词
claude -c                  # 继续上一次会话
claude -r                  # 选择历史会话恢复
claude -p "修复bug"         # 非交互一次性输出（print 模式）
claude --model opus        # 指定模型启动
claude --add-dir ../lib    # 增加可访问目录

3输入框快捷技巧

这些技巧能显著提升你与 Claude Code 的交互效率：

输入作用
!<命令>直接执行 shell 命令（如 !git status）
@<文件路径>引用文件让 Claude 读取
#<内容>快速写入记忆（CLAUDE.md）
Shift + Tab切换自动编辑 / 计划模式
Esc中断 Claude 正在进行的操作
连按两次 Esc回退到上一条消息编辑
Ctrl + C退出 Claude Code
Ctrl + L清屏
↑ / ↓浏览历史输入

4权限与工作模式

  • 默认模式：执行前会请求授权，最安全的工作方式
  • Plan 模式（Shift+Tab 切换）：只规划不改文件，适合需求评审
  • Auto-accept 模式：自动批准编辑操作，适合熟悉的重复性任务
  • Bypass 模式：跳过权限检查，请谨慎使用
小提示：涉及删除、force push、数据库写入等高风险操作时，建议切回默认模式，确保每一步都在掌控之中。

5记忆文件 CLAUDE.md

Claude 会在每次会话自动加载记忆文件，放置位置决定作用范围：

  • ./CLAUDE.md —— 项目级记忆，随项目共享
  • ~/.claude/CLAUDE.md —— 全局用户记忆，所有项目生效
  • ./CLAUDE.local.md —— 本地个人记忆，不提交到 git

可写入项目规范、常用命令、编码约定、架构说明等。

6常用工作流示例

分析并修复 Bug

claude "看看 src/main.cpp 里的崩溃问题并修复"

代码审查与提交

/review                    # 审查当前改动
/commit                    # 生成规范的提交信息

引用文件进行重构

@src/components/Button.tsx 帮我重构这个组件，抽取通用逻辑

恢复之前的会话

claude -c                  # 继续最近一次对话
claude -r                  # 从列表中选择历史会话
效率建议：把常用的 prompt 写进 CLAUDE.md，比如"测试命令用 npm test"、"不要修改生成的文件"等，可以减少重复交代。
